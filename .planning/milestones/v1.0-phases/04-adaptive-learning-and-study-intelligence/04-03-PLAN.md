       - Reusable wrapper card for exercises

       - Cockpit-gray card with navy border, title, brief description, then children content
       - Consistent with existing card patterns in the app

    2. Create `src/components/pbm/DirectionalReasoning.tsx`:
       - Self-contained exercise component with local state (useState/useReducer)
       - Generates random heading problems:
         - "You are flying heading [X]. Turn [left/right] [Y] degrees. What is your new heading?"
         - X: random 0-350 in increments of 10
         - Y: random from [30, 45, 60, 90, 120, 135, 180]
         - Direction: random left/right
       - Shows 4 multiple-choice answers (correct + 3 distractors)
       - Distractors: one with wrong direction, one off by common error (e.g., not wrapping at 360), one random
       - Visual: simple compass rose SVG showing current heading with an arrow
       - On answer: show correct/incorrect with explanation ("270 + 90 = 360 = 0 (North)")
       - Track score: X/Y correct in current session
       - "Next Question" button to generate a new problem
       - "Reset" to clear session score
       - Educational tip at top: "The PBM tests your ability to quickly determine headings and directions. Practice thinking in terms of compass headings."

    3. Create `src/components/pbm/DividedAttention.tsx`:
       - Self-contained exercise with local state and useRef for animation
       - Two simultaneous monitoring tasks:
         - LEFT panel: "Altitude" -- a number that drifts up/down randomly. User must click "Correct" button when it goes above 5000 or below 2000 (out of safe range 2000-5000)
         - RIGHT panel: "Heading" -- a heading value that drifts. User must click "Correct" button when it deviates more than 15 degrees from target heading (shown at top)
       - Values update every 1-2 seconds via setInterval (use useRef for interval ID, cleanup on unmount)
       - Scoring: +1 for each correct detection, -1 for false alarm, -1 for missed detection (value was out of range for 3+ seconds without response)
       - Show current score, time elapsed (30-second rounds)
       - After round ends, show summary: detections, false alarms, misses
       - "Start Round" button, disabled during active round
       - Educational tip: "PBM tests divided attention -- monitoring multiple instruments simultaneously. In this exercise, watch both panels and respond when values go out of range."
       - Use requestAnimationFrame or setInterval (setInterval is simpler and appropriate here since updates are 1-2s)
       - Avoid scope creep: this is NOT a real-time game. Simple value monitoring with click responses.
  </action>
