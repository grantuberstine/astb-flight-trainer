  <name>Task 1: Adaptive types and pure algorithm functions</name>
  <files>src/types/adaptive.ts, src/lib/spaced-repetition.ts, src/lib/adaptive.ts, src/lib/diagnostic.ts, src/lib/study-plan.ts</files>
  <behavior>
    - createCard('q1', 'MST') returns SRCard with easeFactor 2.5, interval 1, repetitions 0, nextReview ~1 day from now
    - reviewCard(card, 1) (fail) resets repetitions to 0, interval to 1, reduces easeFactor by 0.2 (min 1.3)
    - reviewCard(card, 4) (pass) increments repetitions, computes interval based on reps count and easeFactor
    - getDueCards filters to cards where nextReview <= Date.now()
    - computeSectionWeights with all 0-total sections returns weight 1.0 for each
    - computeSectionWeights with 100% accuracy section returns minWeight (0.1 default)
    - computeSectionWeights with 50% accuracy section returns weight 0.5
    - pickWeightedSection returns a valid SectionId based on weight distribution
    - scoreDiagnostic computes per-section accuracy, avgTimeMs, and identifies weakest sections
    - generateStudyPlan with 14 days remaining produces 2 StudyWeek entries
    - generateStudyPlan front-loads weakest sections in early weeks
    - generateStudyPlan makes final week comprehensive review
  </behavior>
  <action>
    1. Create `src/types/adaptive.ts` with all type definitions:
       - `SRCard` (questionId, section, easeFactor, interval, repetitions, nextReview, lastReview -- all numbers except questionId/section strings)
       - `WeightedSection` (sectionId: SectionId, weight: number)
       - `DiagnosticSectionResult` (correct, total, accuracy, avgTimeMs, weakTags: string[])
       - `DiagnosticResult` (timestamp, sectionResults: Record<SectionId, DiagnosticSectionResult>, overallAccuracy, weakestSections: SectionId[], recommendedFocus: SectionId[])
       - `StudyWeek` (weekNumber, startDate: string, endDate: string, focusSections: SectionId[], reviewSections: SectionId[], dailyGoal: number)
       - Use `import type { SectionId }` from question types

    2. Create `src/lib/spaced-repetition.ts` implementing SM-2:
       - `DAY_MS = 86400000` constant
       - `createCard(questionId, section)` -- initial card with easeFactor 2.5, interval 1, nextReview = now + DAY_MS
       - `reviewCard(card, quality)` -- quality 0-5; < 3 fails (reset reps, interval 1, EF -= 0.2 min 1.3); >= 3 passes (reps++, interval by SM-2 formula, EF adjustment)
       - `getDueCards(cards)` -- filter cards where nextReview <= Date.now()
       - `MAX_ACTIVE_CARDS = 30` -- cap for preventing queue explosion (per research pitfall)

    3. Create `src/lib/adaptive.ts`:
       - `computeSectionWeights(scores, minWeight = 0.1)` -- inverse accuracy weighting; untested sections get weight 1.0; formula: Math.max(1 - accuracy, minWeight)
       - `pickWeightedSection(weights)` -- weighted random selection from WeightedSection[]

    4. Create `src/lib/diagnostic.ts`:
       - `DIAGNOSTIC_QUESTIONS_PER_SECTION = 5` (25 total, per research)
       - `scoreDiagnostic(answers: AnswerRecord[])` -- groups answers by section, computes accuracy/avgTimeMs per section, extracts weakTags by looking up question tags from answers, identifies weakest 2-3 sections as recommendedFocus
       - Note: weakTags extraction uses questionId to find the question -- pass a `tagLookup: Record<string, string[]>` parameter so the function stays pure (no async question loading)

    5. Create `src/lib/study-plan.ts`:
       - `generateStudyPlan(testDate: string, sectionWeights: WeightedSection[])` -- divides days remaining into weekly blocks
       - Sort sections by weight descending (weakest first)
       - Early weeks focus on 1-2 weakest sections with strongest as review
       - Final week is comprehensive review across all sections
       - dailyGoal: 20 questions/day base, scaling down to 15 in final week (lighter review)
       - Handle edge cases: test date in past (return 1 comprehensive week), very far out (cap at 12 weeks)
  </action>

  </verify>
