**CockpitView.tsx** -- Canvas 2D component rendering a cockpit instrument view:

Props: `{ pitch: number; bank: number; heading: number; coastline: 'none' | 'left' | 'right' | 'ahead-left' | 'ahead-right'; width?: number; height?: number; className?: string }`

Rendering approach (all in a useEffect with canvas ref):
1. Clear canvas, save context
2. Translate to center, rotate by bank angle (degrees to radians)
3. Calculate horizon offset from pitch: `horizonY = (pitch / 90) * (height / 2)` -- positive pitch moves horizon down (nose up = more sky)
4. Draw sky above horizon: fill with sky blue (#4A90D9 or similar)
5. Draw ground below horizon: fill with earth brown (#6B4226 or similar)
6. Draw horizon line: white, 2px width
7. Draw coastline if not 'none': render as a curved/jagged line at the horizon. For 'left': coastline appears on left side of horizon. For 'right': right side. For 'ahead-left' / 'ahead-right': coastline curves from ahead toward that side. Use a simple bezier curve in earth-brown-to-water-blue transition. Keep it SIMPLE -- readability over realism per user decision.
8. Restore context (un-rotate)
9. Draw fixed aircraft reference symbol at center: small crosshair/wings icon (two short horizontal lines + one vertical line) in white, NOT affected by rotation
10. Apply devicePixelRatio scaling for sharp rendering on retina displays: set canvas width/height attributes to `width * dpr` / `height * dpr`, use CSS width/height for display size, scale context by dpr

Default size: 280x280. Use a container div that can be sized by parent.

**AircraftSilhouette.tsx** -- SVG component showing aircraft from above:

Props: `{ pitch: number; bank: number; heading: number; selected?: boolean; correct?: boolean; incorrect?: boolean; onClick?: () => void; disabled?: boolean; size?: number }`

Render a top-down aircraft silhouette in SVG (viewBox 0 0 60 60):
- Apply `rotate(bank)` transform to the aircraft body group (centered at 30,30)
- Use heading to rotate the entire aircraft: `rotate(heading - 360)` (so north-facing = nose up)
- Pitch shown as vertical offset of the aircraft body

- Border styling: neutral (navy-600) default, gold-400 when selected, green-500 when correct, red-500 when incorrect
- Button wrapper with hover and click states. Disabled state dims opacity.
- Size default 64px

**SAT Question Bank (src/data/questions/sat.ts):**
Author 50+ SpatialQuestion items.

Each question needs:
- `scenarioParams`: the cockpit view the user sees (what pitch/bank/heading/coastline)
- `answerOptions`: 5 aircraft orientations, one matching the cockpit view, four distractors
- `correctAnswer`: index (0-4) of the correct option
- `explanation`: text explaining how to read the cockpit view

**Parameter variety (spread across these combinations):**
- Pitch: -30, -15, 0, 15, 30 (degrees)
- Bank: -45, -30, -15, 0, 15, 30, 45 (degrees)
- Heading: 0, 45, 90, 135, 180, 225, 270, 315 (degrees)
- Coastline: 'none', 'left', 'right', 'ahead-left', 'ahead-right'

**Distractor strategy:** Each wrong answer should differ from correct by ONE parameter (common SAT trap):
- Same pitch/heading but wrong bank
- Same bank/heading but wrong pitch direction
- Same pitch/bank but wrong heading (mirror)
- Coastline mismatch interpretation

**Question structure guidelines:**
- IDs: `sat-001` through `sat-NNN`
- Tags: describe the key challenge, e.g., `['bank-left', 'diving', 'coastline-right']`
- Difficulty: easy = level flight or simple single-axis deviation. Medium = two-axis. Hard = all three axes + coastline.
- ~15 easy, ~25 medium, ~10 hard
- Explanation: describe step-by-step how to read the cockpit view ("The horizon is tilted to the right, indicating a left bank. The horizon is above center, indicating a nose-down attitude...")
