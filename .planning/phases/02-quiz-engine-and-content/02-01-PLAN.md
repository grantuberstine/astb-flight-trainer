**3. Create useQuizSession hook** (src/hooks/useQuizSession.ts):
- useReducer-based state machine with states: `'idle' | 'loading' | 'answering' | 'showing-explanation' | 'complete'`
- State shape: `{ status, questions: Question[], currentIndex, answers: Array<{questionId, selected, correct, timeMs}>, startedAt, questionStartedAt }`
- Actions: `START {sectionId, count, timed, timeLimitSec?}`, `QUESTIONS_LOADED {questions}`, `ANSWER {selected}`, `NEXT`, `QUIT`, `TIME_UP`
- On START: set status to 'loading'. The component will call getQuestions and dispatch QUESTIONS_LOADED.
- On ANSWER: check correctness against current question (handle all 3 question types), record timeMs since questionStartedAt, set status to 'showing-explanation'
- On NEXT: increment currentIndex, if past end set 'complete', else set 'answering' and reset questionStartedAt
- On QUIT: set 'complete' (answers already recorded are kept)
- On TIME_UP: set 'complete'
- Export a `useQuizSession()` hook that returns `{ state, dispatch, currentQuestion, progress: {answered, total, correct} }`
- For PassageQuestion: treat as one "question unit" -- track sub-question index within the reducer state. Add `subQuestionIndex` to state. ANSWER for passage increments sub-question; when all sub-questions done, move to 'showing-explanation' for the last one, then NEXT advances to next question.

**4. Create useCountdown hook** (src/hooks/useCountdown.ts):
- Takes `totalSeconds: number`
- Uses absolute endTime approach (not decrementing counter) to avoid tab-backgrounding drift: store `endTime = Date.now() + totalSeconds * 1000` on start, compute `remaining = Math.max(0, Math.ceil((endTime - Date.now()) / 1000))` on each tick
- Returns `{ remaining, formatted (MM:SS string), isPaused, isExpired, pause(), resume(), reset(newTotal?) }`
- pause() stores elapsed, resume() recalculates endTime from remaining
- 1-second interval ticks, cleans up on unmount

**5. Create question data index** (src/data/questions/index.ts):
- Define lazy loaders map: `Record<SectionId, () => Promise<{default: Question[]}>>` using dynamic `import()` for each section file (mst.ts, rct.ts, mct.ts, anit.ts, sat.ts)
- Export `getQuestions(section: SectionId, count = 10): Promise<Question[]>` -- loads section, shuffles with Fisher-Yates, returns first `count`
